# Celest v0.2.0 手动集成测试方案

> 测试目标：验证 14 个 Bug 修复 + 4 个 Phase 功能增强
> 测试方式：在 VS Code 中打开 Celest 插件，逐一输入提示词，观察结果

---

## 一、环境准备

1. 启动 VS Code，安装本地的 `celest-*.vsix`
2. 确保 CodeWhale TUI 已启动并连接（底部栏显示 🟢 Agent 等状态）
3. 打开一个包含代码文件的工作区（如 celest 项目本身）

---

## 四、@ 文件引用测试（Bug 2）

### 4.1 正则包装行为

| # | 操作/提示词 | 预期结果 |
|---|------------|---------|
| 34 | 输入 `分析 @src/chatViewProvider.ts 的代码` | 自动包装为 `@[src/chatViewProvider.ts]`（含扩展名） |
| 35 | 输入 `请 @model 切换` | **不**自动包装（无扩展名或路径分隔符） |
| 36 | 输入 `查看 @config 设置` | **不**自动包装 |
| 37 | 输入 `编辑 @C:\Users\test\file.txt` | 自动包装为 `@[C:\Users\test\file.txt]`（Windows 路径） |

### 4.2 @ 弹窗行为

| # | 操作 | 预期结果 |
|---|------|---------|
| 38 | 输入 `@` + 等待 300ms + 输入 `cha` | 弹出文件选择弹窗，过滤含 "cha" 的文件 |
| 39 | 仅输入 `@` 不输入后续字符 | 不弹出文件选择弹窗（需要至少 1 个过滤字符） |
| 40 | 输入 `/help @test` | 不弹出 @ 弹窗（斜杠命令上下文中不触发 @） |

### 4.3 文件缺失提示

| # | 提示词 | 预期结果 |
|---|--------|---------|
| 41 | 输入 `分析 @[不存在的文件.txt]` 并发送 | prompt 中应包含 `⚠ 以下文件未找到或无法访问` 提示 |
| 42 | 输入 `分析 @[真实文件.ts]` 并发送 | prompt 中应包含该文件的实际内容，**没有** ⚠ 提示 |

### 4.4 路径穿越防护（Bug 8）

| # | 提示词 | 预期结果 |
|---|--------|---------|
| 43 | 输入 `查看 @[../../../etc/passwd]` 并发送 | prompt 中不应包含系统文件内容，应在缺失提示中报告该路径 |
