## 一、UI 布局

### 1.1 整体布局
- [x] 1.1.1 打开 Celest 面板后，从上到下依次看到：标题栏 → 空状态 → 输入区 → 状态栏
  - 预期：四层垂直排列，无重叠、无溢出
  - 实际：ok
- [x] 1.1.2 面板宽度缩窄至 300px 时，布局不崩坏
  - 预期：输入框自适应缩小，消息文字正常换行
  - 实际：ok

### 1.2 输入区域
- [ ] 1.2.1 输入框最小高度约 60px（3 行），placeholder 文字可见
  - 预期：`Ask anything... (@ to reference files, / for commands)` 完整显示
  - 实际：默认高度可以再调大一点80px
- [ ] 1.2.2 输入多行文本（5 行以上），输入框自动增高但不超过 200px
  - 预期：高度随内容增长，到 200px 后出现内部滚动
  - 实际：增长高度后，输出后，没有恢复到默认高度，shift+enter换行存在多换一行的问题
- [x] 1.2.3 输入区底部显示 "Enter to send, Shift+Enter for newline" 提示
  - 预期：灰色小字，左对齐
  - 实际：ok
- [x] 1.2.4 Send 按钮位于输入区右下角
  - 预期：蓝色主题按钮，输入为空时半透明禁用
  - 实际：ok

### 1.3 Stop 按钮
- [x] 1.3.1 空闲时 Stop 按钮不可见
  - 预期：输入框右上角无红色按钮
  - 实际：ok
- [ ] 1.3.2 生成中 Stop 按钮出现在输入框右上角
  - 预期：红色按钮 `⏹ Stop`，浮动在 textarea 内部右上角
  - 实际：stop应该和send同行，这样保证可以追加队列输入和stop功能不冲突，而且也能明确一些
- [x] 1.3.3 Stop 按钮与 textarea 文字不重叠
  - 预期：textarea 右侧 padding 约 60px，文字在按钮左侧
  - 实际：ok
