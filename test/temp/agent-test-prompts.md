# Agent 测试提示词集

> 用于评估 AI Agent 各项能力的提示词集合，按能力维度分类。

---

## 一、工具调用能力

### 1.1 文件操作

```
在当前目录下创建一个名为 hello.txt 的文件，写入 "Hello World"，然后读取它并打印内容。
```

```
找一下项目中最大的三个文件（按体积），列出它们的路径和大小。
```

```
读取 src/main.rs，把第 15-30 行中的 println! 全部替换为 log::info!。
```

```
把 ./data 目录下所有 .csv 文件合并成一个 all.csv，保留表头。
```

### 1.2 搜索

```
在项目中搜索所有包含 "deprecated" 的代码行，列出文件名和行号。
```

```
在所有 .py 文件中找到定义异步函数的地方（async def），输出函数名和所在文件。
```

```
在 ./src 下找到所有调用 external_api 的地方，分析调用上下文，告诉我这个函数被调用了多少次、分别在什么场景下使用。
```

### 1.3 Shell 执行

```
运行 cargo test，如果失败则分析第一个失败的测试并给出修复建议。
```

```
检查当前系统上 node 和 python 的版本，告诉我哪个项目可能和当前版本不兼容。
```

```
启动一个 HTTP 服务器在端口 8080 上，提供当前目录的静态文件服务。
```

### 1.4 Web 访问

```
访问 https://news.ycombinator.com，提取当前首页的前 10 条新闻标题和链接。
```

```
用百度搜索 "Rust async 最佳实践"，总结前 5 条结果的要点。
```

```
打开 https://github.com/deepseek-ai/DeepSeek-V3/blob/main/README.md，提取模型的关键参数并整理成表格。
```

## 二、代码理解与生成

### 2.1 代码解释

```
解释这个项目中 main.rs 的核心逻辑，用中文说明它启动后做了什么、依赖了哪些外部服务。
```

```
这个函数的复杂度很高，帮我拆解一下它做了什么，并给出重构建议。
```

```
项目中有一个数据流：从 HTTP 请求到数据库写入，画出这个链条涉及的模块和函数调用链。
```

### 2.2 代码生成

```
写一个 Python 函数，从嵌套的 JSON 对象中提取所有值为数字的键值对，扁平化成 { "a.b.c": 123 } 的格式。
```

```
生成一个 Rust 模块，实现一个简单的键值存储引擎，支持 get、set、delete 和 scan(prefix) 操作，数据存储在内存中。
```

```
写一个 GitHub Actions workflow，在 push 到 main 分支时运行测试、构建 Docker 镜像并推送到 ghcr.io。
```

### 2.3 BUG 定位

```
这段代码编译不通过（粘贴错误信息），帮我找出问题在哪。
```

```
项目在压力测试下偶尔崩溃，看看代码中有没有竞态条件或未处理的错误。
```

```
这个 SQL 查询跑得很慢，帮我分析原因并给出优化后的版本。
```

## 三、多步任务拆解

### 3.1 重构

```
把项目中的回调风格代码全部改写成 async/await，不改变外部行为。
```

```
将单体 main.rs 按职责拆分为独立的模块文件，保持所有测试通过。
```

```
把这个 Python 脚本打包成一个可安装的 pip 包，包含 setup.py、入口点、依赖声明。
```

### 3.2 项目初始化

```
初始化一个 Rust CLI 项目，使用 clap 做参数解析，支持三个子命令：init、run、clean，每个子命令有自己的参数。
```

```
在当前目录创建一个前端项目骨架：Vite + React + TypeScript，配置 ESLint 和 Prettier，包含一个基本的计数器组件。
```

### 3.3 数据分析

```
下载这个 CSV 文件（给出 URL），分析各列的分布情况，生成直方图并保存为 png。
```

```
解析项目中的日志文件（路径 ./logs/），统计每个接口的请求次数、平均耗时、P99 延迟，输出 Markdown 表格。
```

## 七、真实场景组合

### 7.1 "帮我热修复一个线上 bug"

```
生产环境反馈用户登录后看到的欢迎名称为空。项目路径 ./myapp，框架是 FastAPI + Jinja2 模板。排查 root cause 并修复。
```

### 7.2 "帮我设置 CI/CD"

```
项目已有一个 Rust workspace（包含 3 个 crate）。配置 GitHub Actions，要求：
- push 和 PR 到 main 时运行 cargo test 和 cargo clippy
- 发布 tag 时构建 Windows / Linux / macOS 二进制并上传到 Release
- 缓存 cargo 依赖以加速构建
```

### 7.3 "帮我生成周报"

```
读取过去 7 天的 git log，按作者分组统计提交次数、增删行数、涉及模块，生成一份 Markdown 周报。项目路径 ./repo。
```

### 7.4 "数据库迁移"

```
项目从 SQLite 迁移到 PostgreSQL。找到所有 SQLite 特有的查询语法，生成迁移脚本，更新 ORM 配置，并说明需要注意的数据类型差异。
```

## 八、边界条件测试

```
对文件执行写操作前先检查它是否存在，如果不存在则报错而不是创建。
```

```
在某个长时间运行的任务中间（一个 shell 命令跑了几分钟），尝试向 Agent 发送第二条指令，观察它能否正确处理中断和优先级。
```

```
同时请求 Agent 做三件不同的事（搜索文件、读取远程网页、运行 shell 命令），观察它是否并行处理。
```

```
说一句和当前任务完全无关的话，测试 Agent 能否保持上下文不丢失。
```

```
连续发送 10 条短消息，每条一个很小的操作（touch file、删除、再创建、改名），测试 Agent 的指令队列处理能力。
```
